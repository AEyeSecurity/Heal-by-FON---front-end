# HEAL Variant Enrichment

This service enriches observed VCF-canon match rows with public external variant sources.

Input:

- `heal_fon_deliverable_presentation_audit.csv` generated by the match preparation step.

Processing:

- Keeps rows where `has_genotype=true`.
- Deduplicates rsIDs for external lookups.
- Uses a local rsID cache to avoid repeating public API calls.
- Queries public sources without API keys:
  - Ensembl Variation
  - Ensembl VEP
  - ClinVar E-utilities
  - MyVariant.info

Outputs:

- `heal_observed_variant_enrichment.csv`
- `observed_variant_enrichment_summary.json`

The enriched CSV includes patient allele context, allele/external support summaries, Ensembl Variation/VEP summaries, ClinVar `esearch` plus `esummary`, MyVariant-derived fields, and compact raw JSON snippets for source-level QA.

Example:

```powershell
python C:\ServerCIT\services\heal-variant-enrichment\enrich_observed_variants.py `
  --input C:\ServerCIT\services\heal-match-preparation\runs\<run-id>\heal_fon_deliverable_presentation_audit.csv `
  --output-dir C:\ServerCIT\services\heal-variant-enrichment\runs\<run-id>
```

The script is designed to degrade safely. If an external source is unavailable, it writes a warning and still produces the audit CSV.
