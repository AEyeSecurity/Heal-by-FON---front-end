# HEAL Match Preparation

This service prepares the VCF-canon match output for audit and downstream interpretation review.
It is called as an internal step of the `HEAL - VCF Canon Match` workflow, not as a separate user-facing pipeline step.

Input:
- `sheet_final_consolidated.csv` generated by the VCF-canon match workflow.

Outputs:
- `heal_fon_deliverable_presentation_min.csv`: compact deliverable-style columns.
- `heal_fon_deliverable_presentation_audit.csv`: expanded audit columns for QA.
- `match_preparation_summary.json`: metadata and count summary.

The service does not parse the original VCF again. It consumes the already matched consolidated CSV, so it is fast and deterministic for the same match run.

Manual test:

```powershell
python C:\ServerCIT\services\heal-match-preparation\prepare_match_deliverable.py `
  --input C:\ServerCIT\services\heal-vcf-canon-match\runs\<run-id>\sheet_final_consolidated.csv `
  --output-dir C:\ServerCIT\services\heal-match-preparation\runs\<run-id>
```

n8n wrapper used by Workflow 4:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass `
  -File C:\ServerCIT\services\heal-match-preparation\run_heal_match_preparation.ps1 `
```
